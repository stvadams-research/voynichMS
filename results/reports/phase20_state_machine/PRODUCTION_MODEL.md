## 2. Lattice Structure

The manuscript's text is generated by traversing a 50-window lattice containing 7717 unique words.

| Property | Value |
|:---|:---|
| Windows | 50 |
| Vocabulary | 7717 words |
| Window size range | 18-396 words |
| Transition type | First-order Markov at window level |
| Canonical admissibility | 64.94% (corrected + suffix recovery) |
| Overgeneration | ~20x at all n-gram orders |

Each word in the codebook maps to a specific next window. When the scribe writes a word, the lattice determines which window to consult next. The per-window correction offset adjusts for systematic drift in the device.

## 3. Production Protocol

### 3.1 Step-by-Step Procedure

**Step 1: INITIALIZE**
: Begin a new line. Set current_window to the window of the previous line's last word, or W18 if starting a new page.

**Step 2: READ STATE**
: Consult the tabula card to read the current window number and its correction offset.

**Step 3: LOOK UP VOCABULARY**
: Open the codebook to the section for the current window. All admissible words for this position are listed there.

**Step 4: SELECT WORD**
: Choose a word from the current window's vocabulary list. Selection is influenced by scribal preference (hand-specific suffix bias and frequency habits).

**Step 5: WRITE WORD**
: Inscribe the chosen word on the manuscript page.

**Step 6: ADVANCE STATE**
: Look up the written word in the lattice map to determine the raw next window number.

**Step 7: APPLY CORRECTION**
: Add the per-window correction offset to the raw window number (modulo 50) to get the corrected next window. This accounts for the device's systematic drift.

**Step 8: REPEAT**
: Return to step 2 with the new current_window. Continue until the line is complete.

### 3.2 Interruption Handling

| Event | Behavior |
|:---|:---|
| LINE BREAK | The current window carries over to the next line. No state reset occurs. |
| PAGE BREAK | The current window MAY reset to W18 (the hub) or carry over. Empirically, W18 dominates all folio starts (100% of 101 folios). |
| SECTION TRANSITION | No device reconfiguration. All sections use the same 50-window lattice and codebook. Thematic variation arises from vocabulary selection within windows, not different device states. |
| SCRIBAL ERROR (SLIP) | The scribe mis-indexes the codebook or misreads the tabula, producing a word from the wrong window. Slips concentrate at W18 (92.6% of observed slips) because W18 has the largest codebook section (396 words across 7+ pages). |
